---
title: 'Paramètres globaux'
description: 'Mintlify vous donne un contrôle total sur l’apparence et le fonctionnement de votre documentation à l’aide du fichier docs.json'
icon: 'gear'
---

Chaque site Mintlify a besoin d’un fichier `docs.json` contenant les paramètres de configuration essentiels. Pour en savoir plus, consultez les [propriétés](#properties) ci-dessous.

<div id="properties">
  ## Propriétés
</div>

<ResponseField name="name" type="string" required>
  Nom de votre projet. Utilisé pour le titre global.

  Exemple : `mintlify`
</ResponseField>

<ResponseField name="navigation" type="Navigation[]" required>
  Un tableau de groupes avec toutes les pages de ce groupe

  <Expandable title="Navigation">
    <ResponseField name="group" type="string">
      Le nom du groupe.

      Exemple : `Settings`
    </ResponseField>

    <ResponseField name="pages" type="string[]">
      Les chemins relatifs vers les fichiers Markdown qui serviront de pages.

      Exemple : `["customization", "page"]`
    </ResponseField>
  </Expandable>
</ResponseField>

<ResponseField name="logo" type="string or object">
  Chemin vers l’image du logo ou objet contenant les chemins vers les logos pour les modes « clair » et « sombre »

  <Expandable title="Logo">
    <ResponseField name="light" type="string">
      Chemin du logo en mode clair
    </ResponseField>

    <ResponseField name="dark" type="string">
      Chemin du logo en mode sombre
    </ResponseField>

    <ResponseField name="href" type="string" default="/">
      Cible du lien lorsqu’on clique sur le logo
    </ResponseField>
  </Expandable>
</ResponseField>

<ResponseField name="favicon" type="string">
  Chemin de l’image du favicon
</ResponseField>

<ResponseField name="colors" type="Colors">
  Codes couleur hexadécimaux pour votre thème global

  <Expandable title="Colors">
    <ResponseField name="primary" type="string" required>
      La couleur principale. Utilisée le plus souvent pour le contenu mis en évidence, les en-têtes de section et les accents, en mode clair
    </ResponseField>

    <ResponseField name="light" type="string">
      La couleur principale pour le mode sombre. Utilisée le plus souvent pour le contenu mis en évidence, les en-têtes de section et les accents, en mode sombre
    </ResponseField>

    <ResponseField name="dark" type="string">
      La couleur principale pour les boutons importants
    </ResponseField>

    <ResponseField name="background" type="object">
      La couleur de l’arrière-plan en mode clair et en mode sombre

      <Expandable title="Object">
        <ResponseField name="light" type="string" required>
          Le code couleur hexadécimal de l’arrière-plan en mode clair
        </ResponseField>

        <ResponseField name="dark" type="string" required>
          Le code couleur hexadécimal de l’arrière-plan en mode sombre
        </ResponseField>
      </Expandable>
    </ResponseField>
  </Expandable>
</ResponseField>

<ResponseField name="topbarLinks" type="TopbarLink[]">
  Tableau des `name` et `url` des liens à inclure dans la barre supérieure

  <Expandable title="TopbarLink">
    <ResponseField name="name" type="string">
      Le nom du bouton.

      Exemple : `Contact us`
    </ResponseField>

    <ResponseField name="url" type="string">
      L’URL vers laquelle vous êtes redirigé après avoir cliqué sur le bouton. Exemple : `https://mintlify.com/docs`
    </ResponseField>
  </Expandable>
</ResponseField>

<ResponseField name="topbarCtaButton" type="Call to Action">
  <Expandable title="Topbar Call to Action">
    <ResponseField name="type" type={'"link" or "github"'} default="link">
      « link » affiche un bouton. « github » affiche les informations du dépôt à l’URL fournie, y compris le nombre d’étoiles GitHub.
    </ResponseField>

    <ResponseField name="url" type="string">
      Si `link` : la cible du bouton.

      Si `github` : le lien vers le dépôt à partir duquel charger les informations GitHub.
    </ResponseField>

    <ResponseField name="name" type="string">
      Texte du bouton. Requis uniquement si `type` est `link`.
    </ResponseField>
  </Expandable>
</ResponseField>

<ResponseField name="versions" type="string[]">
  Tableau des noms de versions. À utiliser uniquement si vous souhaitez afficher différentes versions
  de la documentation avec un menu déroulant dans la barre de navigation.
</ResponseField>

<ResponseField name="anchors" type="Anchor[]">
  Un tableau d’ancres, comprenant les champs `icon`, `color` et `url`.

  <Expandable title="Anchor">
    <ResponseField name="icon" type="string">
      L’icône [Font Awesome](https://fontawesome.com/search?q=heart) utilisée pour mettre en avant l’ancre.

      Exemple : `comments`
    </ResponseField>

    <ResponseField name="name" type="string">
      Le nom du libellé de l’ancre.

      Exemple : `Community`
    </ResponseField>

    <ResponseField name="url" type="string">
      Le début de l’URL qui détermine quelles pages sont incluses dans l’ancre. En général, c’est le nom du dossier où vous placez vos pages.
    </ResponseField>

    <ResponseField name="color" type="string">
      La couleur hex du fond de l’icône de l’ancre. Peut aussi être un dégradé si vous passez un objet avec les propriétés `from` et `to`, chacune étant une couleur hex.
    </ResponseField>

    <ResponseField name="version" type="string">
      À utiliser si vous souhaitez masquer une ancre jusqu’à ce que la version de la documentation appropriée soit sélectionnée.
    </ResponseField>

    <ResponseField name="isDefaultHidden" type="boolean" default="false">
      Passez `true` si vous voulez masquer l’ancre jusqu’à ce que vous redirigiez quelqu’un vers une page de la documentation qui s’y trouve.
    </ResponseField>

    <ResponseField name="iconType" default="duotone" type="string">
      L’un parmi : &quot;brands&quot;, &quot;duotone&quot;, &quot;light&quot;, &quot;sharp-solid&quot;, &quot;solid&quot; ou &quot;thin&quot;
    </ResponseField>
  </Expandable>
</ResponseField>

<ResponseField name="topAnchor" type="Object">
  Remplacez les configurations par défaut de l’ancre la plus haute.

  <Expandable title="Object">
    <ResponseField name="name" default="Documentation" type="string">
      Le nom de l’ancre la plus haute
    </ResponseField>

    <ResponseField name="icon" default="book-open" type="string">
      Icône Font Awesome.
    </ResponseField>

    <ResponseField name="iconType" default="duotone" type="string">
      L’un parmi : &quot;brands&quot;, &quot;duotone&quot;, &quot;light&quot;, &quot;sharp-solid&quot;, &quot;solid&quot; ou &quot;thin&quot;
    </ResponseField>
  </Expandable>
</ResponseField>

<ResponseField name="tabs" type="Tabs[]">
  Un tableau d’onglets de navigation.

  <Expandable title="Tabs">
    <ResponseField name="name" type="string">
      Le nom du libellé de l’onglet.
    </ResponseField>

    <ResponseField name="url" type="string">
      Le début de l’URL qui détermine quelles pages sont incluses dans l’onglet. En général,
      c’est le nom du dossier où vous placez vos pages.
    </ResponseField>
  </Expandable>
</ResponseField>

<ResponseField name="api" type="API">
  Configuration des paramètres d’API. Pour en savoir plus sur les pages d’API, consultez [API Components](/fr/api-playground/demo).

  <Expandable title="API">
    <ResponseField name="baseUrl" type="string">
      L’URL de base pour tous les points de terminaison de l’API. Si `baseUrl` est un tableau, cela activera plusieurs options d’URL de base
      entre lesquelles l’utilisateur peut basculer.
    </ResponseField>

    <ResponseField name="auth" type="Auth">
      <Expandable title="Auth">
        <ResponseField name="method" type="&#x22;bearer&#x22; | &#x22;basic&#x22; | &#x22;key&#x22;">
          La stratégie d’authentification utilisée pour tous les points de terminaison de l’API.
        </ResponseField>

        <ResponseField name="name" type="string">
          Le nom du paramètre d’authentification utilisé dans l’aire de jeu de l’API.

          Si la méthode est `basic`, le format doit être `[usernameName]:[passwordName]`
        </ResponseField>

        <ResponseField name="inputPrefix" type="string">
          La valeur par défaut conçue pour servir de préfixe au champ de saisie de l’authentification.

          Par exemple, si un `inputPrefix` de `AuthKey` est défini, la valeur par défaut du champ d’authentification sera `AuthKey`.
        </ResponseField>
      </Expandable>
    </ResponseField>

    <ResponseField name="playground" type="Playground">
      Configurations pour l’aire de jeu de l’API

      <Expandable title="Playground">
        <ResponseField name="mode" default="show" type="&#x22;show&#x22; | &#x22;simple&#x22; | &#x22;hide&#x22;">
          Indique si l’aire de jeu est affichée, masquée, ou si elle affiche uniquement le point de terminaison sans interactivité utilisateur supplémentaire (`simple`).

          Pour en savoir plus, consultez les [guides du playground](/fr/api-playground/demo)
        </ResponseField>
      </Expandable>
    </ResponseField>

    <ResponseField name="maintainOrder" type="boolean">
      L’activation de cet indicateur garantit que l’ordre des clés dans les pages OpenAPI correspond à l’ordre défini dans le fichier OpenAPI.

      <Warning>Ce comportement sera bientôt activé par défaut, après quoi ce champ sera obsolète.</Warning>
    </ResponseField>
  </Expandable>
</ResponseField>

<ResponseField name="openapi" type="string | string[]">
  Une chaîne ou un tableau de chaînes d’URL ou de chemins relatifs pointant vers votre
  fichier OpenAPI.

  Exemples :

  <CodeGroup>
    ```json Absolute
    "openapi": "https://example.com/openapi.json"
    ```

    ```json Relative
    "openapi": "/openapi.json"
    ```

    ```json Multiple
    "openapi": ["https://example.com/openapi1.json", "/openapi2.json", "/openapi3.json"]
    ```
  </CodeGroup>
</ResponseField>

<ResponseField name="footerSocials" type="FooterSocials">
  Un objet de comptes de réseaux sociaux où la paire clé:propriété représente la plateforme et l’URL du compte.

  Exemple :

  ```json
  {
    "x": "https://x.com/mintlify",
    "website": "https://mintlify.com"
  }
  ```

  <Expandable title="FooterSocials">
    <ResponseField name="[key]" type="string">
      L’une des valeurs suivantes : `website`, `facebook`, `x`, `discord`, `slack`, `github`, `linkedin`, `instagram`, `hacker-news`

      Exemple : `x`
    </ResponseField>

    <ResponseField name="property" type="string">
      L’URL de la plateforme sociale.

      Exemple : `https://x.com/mintlify`
    </ResponseField>
  </Expandable>
</ResponseField>

<ResponseField name="feedback" type="Feedback">
  Configurations pour activer les boutons de retour

  <Expandable title="Feedback">
    <ResponseField name="suggestEdit" type="boolean" default="false">
      Active un bouton permettant aux utilisateurs de proposer des modifications via des pull requests
    </ResponseField>

    <ResponseField name="raiseIssue" type="boolean" default="false">
      Active un bouton permettant aux utilisateurs de signaler un problème concernant la documentation
    </ResponseField>
  </Expandable>
</ResponseField>

<ResponseField name="modeToggle" type="ModeToggle">
  Personnalisez le sélecteur de mode sombre.

  <Expandable title="ModeToggle">
    <ResponseField name="default" type={'"light" or "dark"'}>
      Indique si vous souhaitez toujours afficher le mode clair ou le mode sombre pour les nouveaux utilisateurs. Lorsqu’il n’est pas défini, le mode suit par défaut celui du système d’exploitation de l’utilisateur.
    </ResponseField>

    <ResponseField name="isHidden" type="boolean" default="false">
      Définissez sur true pour masquer le sélecteur clair/sombre. Vous pouvez combiner `isHidden` avec `default` pour forcer votre documentation à utiliser uniquement le mode clair ou le mode sombre. Par exemple :

      <CodeGroup>
        ```json Only Dark Mode
        "modeToggle": {
          "default": "dark",
          "isHidden": true
        }
        ```

        ```json Only Light Mode
        "modeToggle": {
          "default": "light",
          "isHidden": true
        }
        ```
      </CodeGroup>
    </ResponseField>
  </Expandable>
</ResponseField>

<ResponseField name="backgroundImage" type="string">
  Une image d’arrière-plan affichée derrière chaque page. Voir l’exemple avec
  [Infisical](https://infisical.com/docs) et [FRPC](https://frpc.io).
</ResponseField>
