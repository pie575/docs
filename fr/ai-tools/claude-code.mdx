Claude Code est l’outil CLI officiel d’Anthropic. Ce guide vous aidera à configurer Claude Code afin de rédiger et de maintenir votre documentation.

<div id="prerequisites">
  ## Prérequis
</div>

- Abonnement Claude actif (Pro, Max ou accès à l’API)

<div id="setup">
  ## Configuration
</div>

1. Installez Claude Code à l’échelle du système :

```bash
  npm install -g @anthropic-ai/claude-code
```

2. Accédez à votre répertoire de docs.
3. (Facultatif) Ajoutez le fichier `CLAUDE.md` ci-dessous à votre projet.
4. Exécutez `claude` pour démarrer.


<div id="create-claudemd">
  ## Créer `CLAUDE.md`
</div>

Créez un fichier `CLAUDE.md` à la racine de votre dépôt de documentation pour entraîner Claude Code sur vos standards de documentation spécifiques :

```markdown
# Documentation Mintlify

## Relation de travail
- Vous pouvez remettre en question les idées - cela peut mener à une meilleure documentation. Citez vos sources et expliquez votre raisonnement quand vous le faites
- Demandez TOUJOURS des clarifications plutôt que de faire des suppositions
- Ne mentez JAMAIS, ne devinez pas et n'inventez pas d'informations

## Contexte du projet
- Format : fichiers MDX avec YAML frontmatter
- Configuration : docs.json pour la navigation, le thème, les paramètres
- Composants : composants Mintlify

## Stratégie de contenu
- Documentez juste ce qu'il faut pour la réussite de l'utilisateur - ni trop, ni trop peu
- Priorisez la précision et l'utilisabilité de l'information
- Rendez le contenu pérenne quand c'est possible
- Recherchez les informations existantes avant d'ajouter du nouveau contenu. Évitez la duplication sauf si c'est fait pour une raison stratégique
- Vérifiez les modèles existants pour assurer la cohérence
- Commencez par apporter les plus petites modifications raisonnables

## Exigences frontmatter pour les pages
- titre : Titre de page clair et descriptif
- description : Résumé concis pour le SEO/navigation

## Standards de rédaction
- Voix à la deuxième personne (« vous »)
- Prérequis au début du contenu procédural
- Testez tous les exemples de code avant publication
- Respectez le style et le formatage des pages existantes
- Incluez les cas d'usage de base et avancés
- Balises de langage sur tous les blocs de code
- Texte alternatif sur toutes les images
- Chemins relatifs pour les liens internes

## Flux de travail Git
- N'utilisez JAMAIS --no-verify lors des commits
- Demandez comment gérer les modifications non commitées avant de commencer
- Créez une nouvelle branche quand aucune branche claire n'existe pour les modifications
- Commitez fréquemment pendant le développement
- Ne sautez ou ne désactivez JAMAIS les hooks pre-commit

## À ne pas faire
- Omettre le frontmatter sur tout fichier MDX
- Utiliser des URLs absolues pour les liens internes
- Inclure des exemples de code non testés
- Faire des suppositions - demandez toujours des clarifications
```
