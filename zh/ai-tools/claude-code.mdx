Claude Code 是 Anthropic 的官方 CLI 工具。本指南将帮助你完成 Claude Code 的设置，助你编写并维护文档。

<div id="prerequisites">
  ## 先决条件
</div>

- 有效的 Claude 订阅（Pro、Max 或 API（Application Programming Interface，应用程序编程接口）访问权限）

<div id="setup">
  ## 设置
</div>

1. 全局安装 Claude Code：

```bash
  npm install -g @anthropic-ai/claude-code
```

2. 进入您的文档目录。
3. （可选）将下面的 `CLAUDE.md` 文件添加到您的项目中。
4. 运行 `claude` 启动。


<div id="create-claudemd">
  ## 创建 `CLAUDE.md`
</div>

在文档仓库的根目录创建一个 `CLAUDE.md` 文件，用于基于你自己的文档规范对 Claude Code 进行训练：

```markdown
# Mintlify 文档

## 工作关系
- 你可以对想法提出异议——这可以带来更好的文档。当你这样做时，请引用来源并解释你的推理
- 始终要求澄清而不是做假设
- 绝不撒谎、猜测或编造信息

## 项目 context
- 格式：带有 YAML frontmatter 的 MDX 文件
- 配置：用于导航、主题、设置的 docs.json
- 组件：Mintlify 组件

## 内容策略
- 为用户成功记录恰到好处的内容——不多不少
- 优先考虑信息的准确性和可用性
- 尽可能让内容保持长期有效
- 在添加新内容之前搜索现有信息。避免重复，除非出于战略原因
- 检查现有模式以保持一致性
- 从做最小的合理更改开始

## 页面的 frontmatter 要求
- 标题：清晰、描述性的页面标题
- description：用于 SEO/导航的简洁摘要

## 写作标准
- 第二人称语态（"你"）
- 程序性内容开头的先决条件
- 发布前测试所有代码示例
- 匹配现有页面的样式和格式
- 包括基础和高级用例
- 所有代码块上的语言标签
- 所有图片上的替代文本
- 内部链接使用相对路径

## Git 工作流程
- 提交时绝不使用 --no-verify
- 在开始之前询问如何处理未提交的更改
- 当没有明确的分支用于更改时创建新分支
- 在开发过程中频繁提交
- 绝不跳过或禁用预提交钩子

## 不要
- 在任何 MDX 文件上跳过 frontmatter
- 对内部链接使用绝对 URL
- 包含未测试的代码示例
- 做假设——始终要求澄清
```
