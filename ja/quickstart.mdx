<div id="get-started-in-three-steps">
  ## 3ステップで始める
</div>

ドキュメントサイトをローカルで動かし、最初のカスタマイズを行いましょう。

<div id="step-1-set-up-your-local-environment">
  ### 手順 1: ローカル環境をセットアップする
</div>

<AccordionGroup>
  <Accordion icon="copy" title="ドキュメントをローカルにクローンする">
    オンボーディング中に、まだ持っていなければドキュメント用の GitHub リポジトリを作成しています。このリポジトリへのリンクは[ダッシュボード](https://dashboard.mintlify.com)で確認できます。

    リポジトリをローカルにクローンしてドキュメントを編集・プレビューできるようにするには、GitHub Docs の[リポジトリをクローンする](https://docs.github.com/en/repositories/creating-and-managing-repositories/cloning-a-repository)ガイドに従ってください。
  </Accordion>

  <Accordion icon="rectangle-terminal" title="プレビューサーバーを起動する">
    1. Mintlify CLI をインストール: `npm i -g mint`
    2. ドキュメントのディレクトリに移動して実行: `mint dev`
    3. `http://localhost:3000` を開いてドキュメントをライブで確認

    <Tip>ファイルを編集するとプレビューは自動的に更新されます。</Tip>
  </Accordion>
</AccordionGroup>

<div id="step-2-deploy-your-changes">
  ### ステップ 2: 変更をデプロイする
</div>

<AccordionGroup>
  <Accordion icon="github" title="GitHub アプリをインストールする">
    [ダッシュボード](https://dashboard.mintlify.com/settings/organization/github-app)から Mintlify の GitHub アプリをインストールします。

    この GitHub アプリがドキュメントサイトへの変更を自動でデプロイするため、デプロイを手動で管理する必要はありません。
  </Accordion>

  <Accordion icon="palette" title="サイト名と配色を更新する">
    最初の変更として、ドキュメントサイトの名前と配色を更新してみましょう。

    1. エディタで `docs.json` を開きます。
    2. `"name"` フィールドをプロジェクト名に変更します。
    3. `"colors"` をブランドに合わせて更新します。
    4. `http://localhost:3000` で変更を即座に確認できます。

    <Tip>プライマリカラーを変更して、見た目の違いをすぐに確認してみましょう！</Tip>
  </Accordion>
</AccordionGroup>

<div id="step-3-go-live">
  ### ステップ 3: 本番公開
</div>

<Accordion icon="rocket" title="ドキュメントを公開する">
  1. 変更をコミットしてプッシュします。
  2. ほどなくしてドキュメントが更新され、公開されます！
</Accordion>
