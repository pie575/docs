Claude Code は Anthropic の公式 CLI ツールです。本ガイドでは、ドキュメントの作成・保守を効率化するために、Claude Code のセットアップ手順を説明します。

<div id="prerequisites">
  ## 前提条件
</div>

- 有効な Claude サブスクリプション（Pro、Max、または API（Application Programming Interface）アクセス）

<div id="setup">
  ## セットアップ
</div>

1. Claude Code をグローバルにインストールします：

```bash
  npm install -g @anthropic-ai/claude-code
```

2. docs ディレクトリに移動します。
3. （任意）以下の `CLAUDE.md` をプロジェクトに追加します。
4. `claude` を実行して開始します。


<div id="create-claudemd">
  ## `CLAUDE.md` を作成する
</div>

ドキュメントリポジトリのルートに `CLAUDE.md` ファイルを作成し、あなたのドキュメント運用標準に基づいて Claude Code をトレーニングします。

```markdown
# Mintlifyドキュメント

## 作業関係
- アイデアに対して反対意見を述べることができます。これはより良いドキュメントにつながります。その際は情報源を引用し、理由を説明してください
- 推測ではなく、必ず明確化を求めてください
- 嘘をついたり、推測したり、情報を作り上げたりしないでください

## プロジェクトcontext
- 形式：YAML frontmatterを含むMDXファイル
- 設定：ナビゲーション、テーマ、設定用のdocs.json
- コンポーネント：Mintlifyコンポーネント

## コンテンツ戦略
- ユーザーの成功に必要十分なドキュメントを作成する - 多すぎず、少なすぎず
- 情報の正確性と使いやすさを優先する
- 可能な限りコンテンツを長期的に有効にする
- 新しいコンテンツを追加する前に既存の情報を検索する。戦略的な理由がない限り重複を避ける
- 一貫性のために既存のパターンを確認する
- 最小限の合理的な変更から始める

## ページのfrontmatter要件
- title：明確で説明的なページtitle
- description：SEO/ナビゲーション用の簡潔な要約

## 執筆基準
- 二人称（「あなた」）を使用
- 手順的なコンテンツの開始時に前提条件を記載
- 公開前にすべてのコード例をテストする
- 既存ページのスタイルと書式に合わせる
- 基本的な使用例と高度な使用例の両方を含める
- すべてのコードブロックに言語タグを付ける
- すべての画像にalt textを付ける
- 内部リンクには相対パスを使用

## Gitワークフロー
- コミット時に--no-verifyを使用しないでください
- 作業開始前にコミットされていない変更の処理方法を確認する
- 変更に適したブランチが存在しない場合は新しいブランチを作成する
- 開発中は頻繁にコミットする
- pre-commitフックをスキップまたは無効化しないでください

## 禁止事項
- MDXファイルでfrontmatterをスキップしない
- 内部リンクに絶対URLを使用しない
- テストされていないコード例を含めない
- 推測しない - 常に明確化を求める
```
