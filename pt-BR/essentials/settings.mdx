---
title: 'Configurações globais'
description: 'A Mintlify oferece controle total sobre a aparência e a experiência da sua documentação usando o arquivo docs.json'
icon: 'gear'
---

Todo site da Mintlify precisa de um arquivo `docs.json` com as configurações principais. Saiba mais sobre as [propriedades](#properties) abaixo.

<div id="properties">
  ## Propriedades
</div>

<ResponseField name="name" type="string" required>
  Nome do seu projeto. Usado para o título global.

  Exemplo: `mintlify`
</ResponseField>

<ResponseField name="navigation" type="Navigation[]" required>
  Uma lista de grupos com todas as páginas dentro desse grupo

  <Expandable title="Navigation">
    <ResponseField name="group" type="string">
      O nome do grupo.

      Exemplo: `Settings`
    </ResponseField>

    <ResponseField name="pages" type="string[]">
      Os caminhos relativos para os arquivos Markdown que servirão como páginas.

      Exemplo: `["customization", "page"]`
    </ResponseField>
  </Expandable>
</ResponseField>

<ResponseField name="logo" type="string or object">
  Caminho para a imagem do logotipo ou objeto com caminhos para as imagens do logotipo nos modos &quot;light&quot; e &quot;dark&quot;

  <Expandable title="Logo">
    <ResponseField name="light" type="string">
      Caminho para o logotipo no modo claro
    </ResponseField>

    <ResponseField name="dark" type="string">
      Caminho para o logotipo no modo escuro
    </ResponseField>

    <ResponseField name="href" type="string" default="/">
      Para onde o clique no logotipo direciona você
    </ResponseField>
  </Expandable>
</ResponseField>

<ResponseField name="favicon" type="string">
  Caminho para a imagem do favicon
</ResponseField>

<ResponseField name="colors" type="Colors">
  Códigos de cor em hexadecimal para o seu tema global

  <Expandable title="Colors">
    <ResponseField name="primary" type="string" required>
      A cor primária. Usada com mais frequência para conteúdo em destaque, cabeçalhos
      de seção e acentos, no modo claro
    </ResponseField>

    <ResponseField name="light" type="string">
      A cor primária para o modo escuro. Usada com mais frequência para conteúdo
      em destaque, cabeçalhos de seção e acentos, no modo escuro
    </ResponseField>

    <ResponseField name="dark" type="string">
      A cor primária para botões importantes
    </ResponseField>

    <ResponseField name="background" type="object">
      A cor do plano de fundo nos modos claro e escuro

      <Expandable title="Object">
        <ResponseField name="light" type="string" required>
          O código de cor em hexadecimal do plano de fundo no modo claro
        </ResponseField>

        <ResponseField name="dark" type="string" required>
          O código de cor em hexadecimal do plano de fundo no modo escuro
        </ResponseField>
      </Expandable>
    </ResponseField>
  </Expandable>
</ResponseField>

<ResponseField name="topbarLinks" type="TopbarLink[]">
  Lista de `name`s e `url`s dos links que você quer incluir na barra superior

  <Expandable title="TopbarLink">
    <ResponseField name="name" type="string">
      O nome do botão.

      Exemplo: `Contact us`
    </ResponseField>

    <ResponseField name="url" type="string">
      A URL ao clicar no botão. Exemplo: `https://mintlify.com/docs`
    </ResponseField>
  </Expandable>
</ResponseField>

<ResponseField name="topbarCtaButton" type="Call to Action">
  <Expandable title="Topbar Call to Action">
    <ResponseField name="type" type={'"link" or "github"'} default="link">
      &quot;link&quot; mostra um botão. &quot;github&quot; mostra as informações do repositório na URL fornecida, incluindo o número de estrelas no GitHub.
    </ResponseField>

    <ResponseField name="url" type="string">
      Se `link`: para onde o botão direciona.

      Se `github`: link para o repositório de onde carregar as informações do GitHub.
    </ResponseField>

    <ResponseField name="name" type="string">
      Texto dentro do botão. Obrigatório apenas se `type` for `link`.
    </ResponseField>
  </Expandable>
</ResponseField>

<ResponseField name="versions" type="string[]">
  Lista de nomes de versões. Use isto apenas se você quiser mostrar diferentes
  versões da documentação com um menu suspenso na barra de navegação.
</ResponseField>

<ResponseField name="anchors" type="Anchor[]">
  Uma lista de anchors que inclui `icon`, `color` e `url`.

  <Expandable title="Anchor">
    <ResponseField name="icon" type="string">
      O ícone do [Font Awesome](https://fontawesome.com/search?q=heart) usado para destacar o anchor.

      Exemplo: `comments`
    </ResponseField>

    <ResponseField name="name" type="string">
      O nome do rótulo do anchor.

      Exemplo: `Community`
    </ResponseField>

    <ResponseField name="url" type="string">
      O início da URL que determina quais páginas entram no anchor. Geralmente, é o nome da pasta em que você coloca suas páginas.
    </ResponseField>

    <ResponseField name="color" type="string">
      A cor hexadecimal do fundo do ícone do anchor. Também pode ser um gradiente se você passar um objeto com as propriedades `from` e `to`, cada uma sendo uma cor hexadecimal.
    </ResponseField>

    <ResponseField name="version" type="string">
      Use se quiser ocultar um anchor até que a versão correta da documentação seja selecionada.
    </ResponseField>

    <ResponseField name="isDefaultHidden" type="boolean" default="false">
      Passe `true` se quiser ocultar o anchor até você direcionar alguém diretamente para páginas dentro dele.
    </ResponseField>

    <ResponseField name="iconType" default="duotone" type="string">
      Um dentre: &quot;brands&quot;, &quot;duotone&quot;, &quot;light&quot;, &quot;sharp-solid&quot;, &quot;solid&quot; ou &quot;thin&quot;
    </ResponseField>
  </Expandable>
</ResponseField>

<ResponseField name="topAnchor" type="Object">
  Substitua as configurações padrão do anchor superior.

  <Expandable title="Object">
    <ResponseField name="name" default="Documentation" type="string">
      O nome do anchor superior
    </ResponseField>

    <ResponseField name="icon" default="book-open" type="string">
      Ícone do Font Awesome.
    </ResponseField>

    <ResponseField name="iconType" default="duotone" type="string">
      Um dentre: &quot;brands&quot;, &quot;duotone&quot;, &quot;light&quot;, &quot;sharp-solid&quot;, &quot;solid&quot; ou &quot;thin&quot;
    </ResponseField>
  </Expandable>
</ResponseField>

<ResponseField name="tabs" type="Tabs[]">
  Uma lista de abas de navegação.

  <Expandable title="Tabs">
    <ResponseField name="name" type="string">
      O nome do rótulo da aba.
    </ResponseField>

    <ResponseField name="url" type="string">
      O início da URL que determina quais páginas entram na aba. Geralmente, este
      é o nome da pasta em que você coloca suas páginas.
    </ResponseField>
  </Expandable>
</ResponseField>

<ResponseField name="api" type="API">
  Configuração das definições da API. Saiba mais sobre páginas de API em [API Components](/pt-BR/api-playground/demo).

  <Expandable title="API">
    <ResponseField name="baseUrl" type="string">
      A URL base para todos os endpoints da API. Se `baseUrl` for um array, isso habilita várias opções de URL base
      que o usuário pode alternar.
    </ResponseField>

    <ResponseField name="auth" type="Auth">
      <Expandable title="Auth">
        <ResponseField name="method" type="&#x22;bearer&#x22; | &#x22;basic&#x22; | &#x22;key&#x22;">
          A estratégia de autenticação usada para todos os endpoints da API.
        </ResponseField>

        <ResponseField name="name" type="string">
          O nome do parâmetro de autenticação usado no playground da API.

          Se o método for `basic`, o formato deve ser `[usernameName]:[passwordName]`
        </ResponseField>

        <ResponseField name="inputPrefix" type="string">
          O valor padrão projetado para servir como prefixo do campo de entrada de autenticação.

          Ex.: um `inputPrefix` de `AuthKey` fará com que o valor padrão do campo de autenticação seja `AuthKey`.
        </ResponseField>
      </Expandable>
    </ResponseField>

    <ResponseField name="playground" type="Playground">
      Configurações do playground da API

      <Expandable title="Playground">
        <ResponseField name="mode" default="show" type="&#x22;show&#x22; | &#x22;simple&#x22; | &#x22;hide&#x22;">
          Define se o playground está visível, oculto ou exibindo apenas o endpoint, sem interatividade adicional do usuário (`simple`).

          Saiba mais nos [guias do playground](/pt-BR/api-playground/demo)
        </ResponseField>
      </Expandable>
    </ResponseField>

    <ResponseField name="maintainOrder" type="boolean">
      Ativar este sinalizador garante que a ordenação das chaves nas páginas OpenAPI corresponda à ordenação definida no arquivo OpenAPI.

      <Warning>Esse comportamento em breve será ativado por padrão; quando isso ocorrer, este campo será descontinuado.</Warning>
    </ResponseField>
  </Expandable>
</ResponseField>

<ResponseField name="openapi" type="string | string[]">
  Uma string ou um array de strings com URLs ou caminhos relativos que apontam para o seu
  arquivo OpenAPI.

  Exemplos:

  <CodeGroup>
    ```json Absolute
    "openapi": "https://example.com/openapi.json"
    ```

    ```json Relative
    "openapi": "/openapi.json"
    ```

    ```json Multiple
    "openapi": ["https://example.com/openapi1.json", "/openapi2.json", "/openapi3.json"]
    ```
  </CodeGroup>
</ResponseField>

<ResponseField name="footerSocials" type="FooterSocials">
  Um objeto de contas de mídia social em que o par chave:propriedade representa a plataforma e a URL da conta.

  Exemplo:

  ```json
  {
    "x": "https://x.com/mintlify",
    "website": "https://mintlify.com"
  }
  ```

  <Expandable title="FooterSocials">
    <ResponseField name="[key]" type="string">
      Um dos seguintes valores: `website`, `facebook`, `x`, `discord`, `slack`, `github`, `linkedin`, `instagram`, `hacker-news`

      Exemplo: `x`
    </ResponseField>

    <ResponseField name="property" type="string">
      A URL da plataforma social.

      Exemplo: `https://x.com/mintlify`
    </ResponseField>
  </Expandable>
</ResponseField>

<ResponseField name="feedback" type="Feedback">
  Configurações para habilitar botões de feedback

  <Expandable title="Feedback">
    <ResponseField name="suggestEdit" type="boolean" default="false">
      Habilita um botão para permitir que os usuários sugiram edições via pull requests
    </ResponseField>

    <ResponseField name="raiseIssue" type="boolean" default="false">
      Habilita um botão para permitir que os usuários registrem um problema sobre a documentação
    </ResponseField>
  </Expandable>
</ResponseField>

<ResponseField name="modeToggle" type="ModeToggle">
  Personalize o toggle de modo escuro.

  <Expandable title="ModeToggle">
    <ResponseField name="default" type={'"light" or "dark"'}>
      Defina se você sempre quer exibir o modo claro ou escuro para novos usuários. Quando não
      definido, o padrão segue o mesmo modo do sistema operacional do usuário.
    </ResponseField>

    <ResponseField name="isHidden" type="boolean" default="false">
      Defina como true para ocultar o toggle de modo claro/escuro. Você pode combinar `isHidden` com `default` para forçar sua documentação a usar apenas o modo claro ou escuro. Por exemplo:

      <CodeGroup>
        ```json Only Dark Mode
        "modeToggle": {
          "default": "dark",
          "isHidden": true
        }
        ```

        ```json Only Light Mode
        "modeToggle": {
          "default": "light",
          "isHidden": true
        }
        ```
      </CodeGroup>
    </ResponseField>
  </Expandable>
</ResponseField>

<ResponseField name="backgroundImage" type="string">
  Uma imagem de fundo exibida atrás de todas as páginas. Veja o exemplo com
  [Infisical](https://infisical.com/docs) e [FRPC](https://frpc.io).
</ResponseField>
