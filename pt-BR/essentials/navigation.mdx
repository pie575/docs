O menu de navegação é a lista de links presente em todo site.

Você provavelmente vai atualizar o `docs.json` sempre que adicionar uma nova página. As páginas não aparecem automaticamente.

<div id="navigation-syntax">
  ## Sintaxe de navegação
</div>

Nossa sintaxe de navegação é recursiva, o que significa que você pode criar grupos de navegação em níveis (aninhados). Você não precisa incluir `.mdx` nos nomes das páginas.

<CodeGroup>

```json Regular Navigation
"navigation": {
  "tabs": [
    {
      "tab": "Docs",
      "groups": [
        {
          "group": "Getting Started",
          "pages": ["quickstart"]
        }
      ]
    }
  ]
}
```

```json Nested Navigation
"navigation": {
  "tabs": [
    {
      "tab": "Docs",
      "groups": [
        {
          "group": "Getting Started",
          "pages": [
            "quickstart",
            {
              "group": "Nested Reference Pages",
              "pages": ["nested-reference-page"]
            }
          ]
        }
      ]
    }
  ]
}
```

</CodeGroup>

<div id="folders">
  ## Pastas
</div>

Basta colocar seus arquivos MDX em pastas e atualizar os caminhos em `docs.json`.

Por exemplo, para ter uma página em `https://yoursite.com/your-folder/your-page`, você criaria uma pasta chamada `your-folder` contendo um arquivo MDX chamado `your-page.mdx`.

<Warning>
  Você não pode usar `api` como nome de pasta, a menos que a coloque dentro de outra pasta. A Mintlify usa o Next.js, que reserva a pasta `api` no nível raiz para chamadas internas do servidor. Um nome de pasta como `api-reference` seria aceito.
</Warning>

```json Navigation With Folder
"navigation": {
  "tabs": [
    {
      "tab": "Docs",
      "groups": [
        {
          "group": "Nome do Grupo",
          "pages": ["your-folder/your-page"]
        }
      ]
    }
  ]
}
```


<div id="hidden-pages">
  ## Páginas ocultas
</div>

Arquivos MDX que não estão incluídos em `docs.json` não aparecerão na barra lateral, mas podem ser acessados pela barra de pesquisa e por links diretos.
