---
title: 'Configuración global'
description: 'Mintlify te ofrece control total sobre el aspecto y la experiencia de tu documentación mediante el archivo docs.json'
icon: 'gear'
---

Cada sitio de Mintlify necesita un archivo `docs.json` con la configuración principal. Obtén más información sobre las [propiedades](#properties) a continuación.

<div id="properties">
  ## Propiedades
</div>

<ResponseField name="name" type="string" required>
  Nombre de tu proyecto. Se usa para el título global.

  Ejemplo: `mintlify`
</ResponseField>

<ResponseField name="navigation" type="Navigation[]" required>
  Un array de grupos con todas las páginas dentro de ese grupo

  <Expandable title="Navigation">
    <ResponseField name="group" type="string">
      El nombre del grupo.

      Ejemplo: `Settings`
    </ResponseField>

    <ResponseField name="pages" type="string[]">
      Las rutas relativas a los archivos Markdown que se usarán como páginas.

      Ejemplo: `["customization", "page"]`
    </ResponseField>
  </Expandable>
</ResponseField>

<ResponseField name="logo" type="string or object">
  Ruta a la imagen del logotipo u objeto con la ruta a las imágenes del logotipo para los modos &quot;light&quot; y &quot;dark&quot;

  <Expandable title="Logo">
    <ResponseField name="light" type="string">
      Ruta al logotipo en modo claro
    </ResponseField>

    <ResponseField name="dark" type="string">
      Ruta al logotipo en modo oscuro
    </ResponseField>

    <ResponseField name="href" type="string" default="/">
      A dónde te lleva al hacer clic en el logotipo
    </ResponseField>
  </Expandable>
</ResponseField>

<ResponseField name="favicon" type="string">
  Ruta a la imagen del favicon
</ResponseField>

<ResponseField name="colors" type="Colors">
  Códigos de color hexadecimales para tu tema global

  <Expandable title="Colors">
    <ResponseField name="primary" type="string" required>
      El color principal. Se usa con mayor frecuencia para contenido destacado,
      encabezados de sección y acentos en modo claro
    </ResponseField>

    <ResponseField name="light" type="string">
      El color principal para el modo oscuro. Se usa con mayor frecuencia para
      contenido destacado, encabezados de sección y acentos en modo oscuro
    </ResponseField>

    <ResponseField name="dark" type="string">
      El color principal para botones importantes
    </ResponseField>

    <ResponseField name="background" type="object">
      El color del fondo tanto en modo claro como en modo oscuro

      <Expandable title="Object">
        <ResponseField name="light" type="string" required>
          El código de color hex del fondo en modo claro
        </ResponseField>

        <ResponseField name="dark" type="string" required>
          El código de color hex del fondo en modo oscuro
        </ResponseField>
      </Expandable>
    </ResponseField>
  </Expandable>
</ResponseField>

<ResponseField name="topbarLinks" type="TopbarLink[]">
  Array de `name`s y `url`s de los enlaces que quieres incluir en la barra superior

  <Expandable title="TopbarLink">
    <ResponseField name="name" type="string">
      El nombre del botón.

      Ejemplo: `Contact us`
    </ResponseField>

    <ResponseField name="url" type="string">
      La URL a la que se accede al hacer clic en el botón. Ejemplo: `https://mintlify.com/docs`
    </ResponseField>
  </Expandable>
</ResponseField>

<ResponseField name="topbarCtaButton" type="Call to Action">
  <Expandable title="Topbar Call to Action">
    <ResponseField name="type" type={'"link" or "github"'} default="link">
      &quot;link&quot; muestra un botón. &quot;github&quot; muestra la información del repositorio en la URL proporcionada, incluido el número de estrellas de GitHub.
    </ResponseField>

    <ResponseField name="url" type="string">
      Si `link`: a qué enlaza el botón.

      Si `github`: enlace al repositorio del cual cargar la información de GitHub.
    </ResponseField>

    <ResponseField name="name" type="string">
      Texto del botón. Solo es obligatorio si `type` es `link`.
    </ResponseField>
  </Expandable>
</ResponseField>

<ResponseField name="versions" type="string[]">
  Array de nombres de versión. Úsalo solo si quieres mostrar diferentes versiones
  de la documentación con un menú desplegable en la barra de navegación.
</ResponseField>

<ResponseField name="anchors" type="Anchor[]">
  Un arreglo de anchors que incluye `icon`, `color` y `url`.

  <Expandable title="Anchor">
    <ResponseField name="icon" type="string">
      El icono de [Font Awesome](https://fontawesome.com/search?q=heart) usado para destacar el anchor.

      Ejemplo: `comments`
    </ResponseField>

    <ResponseField name="name" type="string">
      El nombre de la etiqueta del anchor.

      Ejemplo: `Community`
    </ResponseField>

    <ResponseField name="url" type="string">
      El inicio de la URL que determina qué páginas pertenecen al anchor. Generalmente, es el nombre de la carpeta donde colocas tus páginas.
    </ResponseField>

    <ResponseField name="color" type="string">
      El color hex del fondo del icono del anchor. También puede ser un degradado si pasas un objeto con las propiedades `from` y `to`, cada una con un color hex.
    </ResponseField>

    <ResponseField name="version" type="string">
      Úsalo si quieres ocultar un anchor hasta que se seleccione la versión correcta de la documentación.
    </ResponseField>

    <ResponseField name="isDefaultHidden" type="boolean" default="false">
      Pasa `true` si quieres ocultar el anchor hasta que enlaces directamente a alguien a la documentación dentro de él.
    </ResponseField>

    <ResponseField name="iconType" default="duotone" type="string">
      Uno de: &quot;brands&quot;, &quot;duotone&quot;, &quot;light&quot;, &quot;sharp-solid&quot;, &quot;solid&quot; o &quot;thin&quot;
    </ResponseField>
  </Expandable>
</ResponseField>

<ResponseField name="topAnchor" type="Object">
  Sobrescribe la configuración predeterminada para el anchor superior.

  <Expandable title="Object">
    <ResponseField name="name" default="Documentation" type="string">
      El nombre del anchor superior
    </ResponseField>

    <ResponseField name="icon" default="book-open" type="string">
      Icono de Font Awesome.
    </ResponseField>

    <ResponseField name="iconType" default="duotone" type="string">
      Uno de: &quot;brands&quot;, &quot;duotone&quot;, &quot;light&quot;, &quot;sharp-solid&quot;, &quot;solid&quot; o &quot;thin&quot;
    </ResponseField>
  </Expandable>
</ResponseField>

<ResponseField name="tabs" type="Tabs[]">
  Un arreglo de pestañas de navegación.

  <Expandable title="Tabs">
    <ResponseField name="name" type="string">
      El nombre de la etiqueta de la pestaña.
    </ResponseField>

    <ResponseField name="url" type="string">
      El inicio de la URL que determina qué páginas pertenecen a la pestaña. Generalmente,
      es el nombre de la carpeta donde colocas tus páginas.
    </ResponseField>
  </Expandable>
</ResponseField>

<ResponseField name="api" type="API">
  Configuración de la API. Obtén más información sobre las páginas de API en [API Components](/es/api-playground/demo).

  <Expandable title="API">
    <ResponseField name="baseUrl" type="string">
      La URL base para todos los endpoints de la API. Si `baseUrl` es un array, permitirá múltiples opciones de URL base
      que el usuario puede alternar.
    </ResponseField>

    <ResponseField name="auth" type="Auth">
      <Expandable title="Auth">
        <ResponseField name="method" type="&#x22;bearer&#x22; | &#x22;basic&#x22; | &#x22;key&#x22;">
          La estrategia de autenticación utilizada para todos los endpoints de la API.
        </ResponseField>

        <ResponseField name="name" type="string">
          El nombre del parámetro de autenticación utilizado en el playground de la API.

          Si el método es `basic`, el formato debe ser `[usernameName]:[passwordName]`
        </ResponseField>

        <ResponseField name="inputPrefix" type="string">
          El valor predeterminado diseñado como prefijo para el campo de entrada de autenticación.

          Por ejemplo, un `inputPrefix` de `AuthKey` hará que el resultado de entrada predeterminado del campo de autenticación sea `AuthKey`.
        </ResponseField>
      </Expandable>
    </ResponseField>

    <ResponseField name="playground" type="Playground">
      Configuración del playground de la API

      <Expandable title="Playground">
        <ResponseField name="mode" default="show" type="&#x22;show&#x22; | &#x22;simple&#x22; | &#x22;hide&#x22;">
          Indica si el playground se muestra, está oculto o solo presenta el endpoint sin interactividad adicional del usuario (`simple`).

          Más información en las [guías del playground](/es/api-playground/demo)
        </ResponseField>
      </Expandable>
    </ResponseField>

    <ResponseField name="maintainOrder" type="boolean">
      Al habilitar este indicador, el orden de las claves en las páginas de OpenAPI coincidirá con el orden definido en el archivo de OpenAPI.

      <Warning>Este comportamiento pronto se habilitará de forma predeterminada, momento en el cual este campo quedará obsoleto.</Warning>
    </ResponseField>
  </Expandable>
</ResponseField>

<ResponseField name="openapi" type="string | string[]">
  Una cadena o un array de cadenas con URL o rutas relativas que apuntan a tu
  archivo de OpenAPI.

  Ejemplos:

  <CodeGroup>
    ```json Absolute
    "openapi": "https://example.com/openapi.json"
    ```

    ```json Relative
    "openapi": "/openapi.json"
    ```

    ```json Multiple
    "openapi": ["https://example.com/openapi1.json", "/openapi2.json", "/openapi3.json"]
    ```
  </CodeGroup>
</ResponseField>

<ResponseField name="footerSocials" type="FooterSocials">
  Un objeto de cuentas de redes sociales donde el par clave:propiedad representa la plataforma y la URL de la cuenta.

  Ejemplo:

  ```json
  {
    "x": "https://x.com/mintlify",
    "website": "https://mintlify.com"
  }
  ```

  <Expandable title="FooterSocials">
    <ResponseField name="[key]" type="string">
      Uno de los siguientes valores: `website`, `facebook`, `x`, `discord`, `slack`, `github`, `linkedin`, `instagram`, `hacker-news`

      Ejemplo: `x`
    </ResponseField>

    <ResponseField name="property" type="string">
      La URL de la plataforma social.

      Ejemplo: `https://x.com/mintlify`
    </ResponseField>
  </Expandable>
</ResponseField>

<ResponseField name="feedback" type="Feedback">
  Configuración para habilitar botones de comentarios

  <Expandable title="Feedback">
    <ResponseField name="suggestEdit" type="boolean" default="false">
      Habilita un botón para que los usuarios sugieran ediciones mediante pull requests
    </ResponseField>

    <ResponseField name="raiseIssue" type="boolean" default="false">
      Habilita un botón para que los usuarios reporten un problema sobre la documentación
    </ResponseField>
  </Expandable>
</ResponseField>

<ResponseField name="modeToggle" type="ModeToggle">
  Personaliza el interruptor del modo oscuro.

  <Expandable title="ModeToggle">
    <ResponseField name="default" type={'"light" or "dark"'}>
      Define si quieres mostrar siempre el modo claro u oscuro a los usuarios nuevos. Si no se define, usaremos por defecto el mismo modo que el sistema operativo del usuario.
    </ResponseField>

    <ResponseField name="isHidden" type="boolean" default="false">
      Establece en true para ocultar el interruptor de modo claro/oscuro. Puedes combinar `isHidden` con `default` para forzar que tu documentación use solo el modo claro u oscuro. Por ejemplo:

      <CodeGroup>
        ```json Only Dark Mode
        "modeToggle": {
          "default": "dark",
          "isHidden": true
        }
        ```

        ```json Only Light Mode
        "modeToggle": {
          "default": "light",
          "isHidden": true
        }
        ```
      </CodeGroup>
    </ResponseField>
  </Expandable>
</ResponseField>

<ResponseField name="backgroundImage" type="string">
  Una imagen de fondo que se mostrará detrás de cada página. Consulta el ejemplo con
  [Infisical](https://infisical.com/docs) y [FRPC](https://frpc.io).
</ResponseField>
